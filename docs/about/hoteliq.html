The problem

Hotels and hospitality intermediaries in Colombia were making pricing decisions based on manual lookups and gut instinct. Pulling competitive pricing data for a single city took a trained operator roughly 250 minutes and produced around 50 data points — all of which were already stale by the time the analysis was done.

A hospitality intermediary needed a tool that could automate this process and turn it into actionable intelligence. They used it in two ways: to negotiate favorable bulk rates with hotels — knowing what the market actually charges is leverage — and to advise their hotel clients on competitive pricing, showing where a property stood relative to the market, city by city, date by date.

What was built

HotelIQ is an on-demand hotel market intelligence platform that collects live competitor pricing data and surfaces it in an interactive dashboard — triggered by a non-technical operator filling out a single web form.

Result: 250 minutes of manual work replaced by a 5-minute automated run, with up to 100 pricing records per query instead of 50.

Skills demonstrated

End-to-end system design

Designed a pipeline connecting a browser interface to an extraction service to a storage layer to a BI dashboard — coordinating across multiple platforms with defined contracts between each of six architectural layers.

Problem scoping & vendor evaluation

Spent a week researching whether an official data partnership was possible before writing pipeline code. Found the team's existing extraction tool had been sandboxed by the target platform — silently returning fabricated prices. Weighed rebuilding it, a third-party vendor (minimum volume too large), and a managed service that fit the constraints.

Constraint-driven infrastructure

With no dedicated server budget and a need to deliver quickly, built the entire backend on zero-provisioning services. Evaluated a traditional stack (in-memory cache + relational database) and ruled it out — operational overhead was disproportionate to the project's scale.

Reliability without a backend

Implemented chunked data delivery with exponential-backoff retry on the pipeline side, and chunk-level idempotency with a distributed concurrency lock on the receiving side — so no data is lost or duplicated even when requests fail or concurrent triggers fire.

Accessible operator interface

Designed the operator UI to require zero training: a single form, immediate status feedback, abuse prevention via cooldown, and the dashboard embedded on the same page.

Key decisions

DecisionWhat was consideredWhy
Managed extraction service Custom scraper · third-party vendor Custom scraper was sandboxed; vendor minimum exceeded project scale.
Cloud-native storage In-memory cache + relational database No infrastructure budget; fits existing client ecosystem.
On-demand cloud compute Dedicated server No idle cost; spins up per request; nothing to maintain.
Serverless gateway for auth Credentials through the browser Credentials in client-side code are a security vulnerability.
Browser-compatible request technique Standard async HTTP Gateway CORS restrictions required an alternative approach.
